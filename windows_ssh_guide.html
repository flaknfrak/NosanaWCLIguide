NOSANA
Windows SSH Setup Guide
Configure SSH access to your Windows Nosana node for remote management
‚Üê Back to Main Guide
All commands must be run in PowerShell as Administrator on your Windows Nosana Node machine
1.
Install OpenSSH Server
Run in Administrator PowerShell on Windows
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
2.
Start SSH Service
Start the SSH daemon service
Start-Service sshd
3.
Verify SSH Service Status
Should show "Running" status
Get-Service sshd
Expected output: Status should show "Running"
4.
Set SSH to Auto-Start
SSH service will start automatically on boot
Set-Service -Name sshd -StartupType 'Automatic'
5.
Check Firewall Rule
Verify SSH firewall rule is enabled
Get-NetFirewallRule -Name "OpenSSH-Server-In-TCP"
Rule should be enabled. If not found, Windows will create it automatically.
6.
Check Current Username
Copy your username. It will look like "desktop-slnkd2r\nosananode"
whoami
Save this username - you'll need it for SSH connections!
7.
Check IP Address
Copy the IPv4 address from the output
ipconfig
Look for "IPv4 Address" under your active network adapter. Save this IP address!
8.
Set User Password
Replace "username" with your actual username (e.g., net user "nosananode" *). Keep the quotation marks.
net user "username" *
Password won't be visible while typing. Remember this password for SSH access!
9.
Test SSH Connection
Test from another machine. Type "yes" to accept host key, then enter password.
ssh "NosanaNode1@110.0.0.299"
Replace "NosanaNode1" with your username and "110.0.0.299" with your actual IP address!
If successful, you'll see a command prompt from your Windows machine!
