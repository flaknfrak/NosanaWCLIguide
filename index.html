Initial Setup
⚠️ IMPORTANT FIRST STEP
Change DNS to 8.8.8.8 and 8.8.4.4 in network adapter IPv4 DNS settings, then restart your computer.
Configure DNS Settings
Set DNS to 8.8.8.8 and 8.8.4.4 in network adapter IPv4 settings
Restart Computer
Restart after DNS configuration

Ubuntu 22.04 LTS Installation
Open PowerShell as Administrator
wsl --install -d Ubuntu-22.04
Start Ubuntu and Verify Installation (Can use start menu to click Ubuntu)
lsb_release -a
Ubuntu is now added to start menu (WSL)
⚙️
WSL Configuration (File creation)
Create .wslconfig File
Create in C:\Users\[USERNAME]\ - Use notepad, save as ".wslconfig" with file type "all"
Configure .wslconfig Content
[wsl2] memory=58GB # ~90% of your RAM processors=16 # Match your CPU cores swap=16GB
Modify values according to your system specs
Verify Memory Configuration
free -h
Run in Ubuntu to check memory allocation and close Ubuntu after check
🐳
Docker Installation (AMD64 version)
Download Docker Desktop
From https://www.docker.com/products/docker-desktop/
Install Docker with WSL 2
Check "Use WSL 2 instead of Hyper-V" during installation. Windows will log out - log back in after installation
Login with your email and use personal style
Configure Docker Settings
Settings → General → Enable "Use WSL 2 based engine"
Enable WSL Integration
Settings → Resources → WSL Integration → Enable Ubuntu-22.04
Add User to Docker Group
sudo usermod -aG docker $USER
Close Ubuntu window, wait 10 seconds, then reopen
Test Docker Installation
docker run hello-world
PC Restart (Optional)
I restarted the PC at this step. May not need to but I did

NVIDIA Container Setup (In Ubuntu)
Check NVIDIA Drivers
nvidia-smi
Update to latest drivers with Windows... Go to the site download the driver and update like normal if needed
Add NVIDIA Repository
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg && curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list && sudo apt-get update
Install NVIDIA Container Toolkit
sudo apt-get install -y nvidia-container-toolkit
Configure Container
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
Verify CDI Configuration
sudo nvidia-ctk cdi list

Podman v4 Installation (In Ubuntu)
Add Podman Repository
echo "deb https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/ /" | sudo tee /etc/apt/sources.list.d/devel:kubic:libcontainers:unstable.list
Add Repository Key
curl -fsSL https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/Release.key | sudo gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/devel_kubic_libcontainers_unstable.gpg > /dev/null
Update and Install Podman
sudo apt update && sudo apt install podman
Check Podman Version
podman --version
Test GPU Access with Podman
podman run --rm --device nvidia.com/gpu=all --security-opt=label=disable ubuntu nvidia-smi -L
Look for: GPU 0: NVIDIA GeForce RTX... with UUID

Nosana Installation (In Ubuntu)
Run Nosana Installation Script
bash <(wget -qO- https://nosana.com/start.sh)

Wallet Management
🔒 CRITICAL: Backup Your Keys Before Funding!
Always backup your private key before adding funds to your wallet.
View & Backup Private Key
sudo cat ~/.nosana/nosana_key.json
Copy the long string with [ ] brackets and backup securely
Import to Phantom Wallet
Use "Import Private Key" in Phantom with your backed up key
Replace Private Key (if importing existing wallet)
sudo nano ~/.nosana/nosana_key.json
Clear with Ctrl+K, paste new key with [ ] brackets, save with Ctrl+X, Y, Enter

Configure DNS Settings in WSL (Manual DNS set)
Create/Edit WSL Configuration File
sudo nano /etc/wsl.conf
Run in Ubuntu
Add Network Configuration
[network] generateResolvConf = false
Add this to the file, save with Ctrl+X, Y, Enter
Close All WSL Terminals
Close all WSL terminals/sessions before shutdown
Shutdown WSL
wsl --shutdown
Run in Windows Command Prompt or PowerShell as Administrator
Open WSL Terminal Again
Start Ubuntu after WSL shutdown
Configure DNS Servers
sudo nano /etc/resolv.conf
Run in Ubuntu
Add DNS Nameservers
nameserver 8.8.8.8 nameserver 8.8.4.4
Add these lines to the file, save with Ctrl+X, Y, Enter

SSH Setup (Windows)
Install OpenSSH Server (Nosana Node but use PowerShell)
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Run in Administrator PowerShell on Windows
Start SSH Service (Nosana Node but use PowerShell)
Start-Service sshd
Start the SSH daemon service
Verify SSH Service Status (Nosana Node but use PowerShell)
Get-Service sshd
Should show "Running" status
Set SSH to Auto-Start (Nosana Node but use PowerShell)
Set-Service -Name sshd -StartupType 'Automatic'
SSH service will start automatically on boot
Check Firewall Rule (Nosana Node but use PowerShell)
Get-NetFirewallRule -Name "OpenSSH-Server-In-TCP"
Verify SSH firewall rule is enabled
Check Current Username (Nosana Node but use PowerShell)
whoami
Copy your username. It will look like "desktop-slnkd2r\nosananode" (or however you named your Windows)
Check IP Address (Nosana Node but use PowerShell)
ipconfig
Copy the IPv4 address from the output
Set User Password (Nosana Node but use PowerShell)
net user "username" *
Replace "username" with your actual username (e.g., net user "nosananode" *). Keep the quotation marks. Password won't be visible while typing
Test SSH Connection (From the Monitor pc in CMD)
ssh "NosanaNode1@110.0.0.299"
Test from another machine. Type "yes" to accept host key, then enter password. *Note: ensure to replace your username and IP address
💻
Create a .bat file to SSH into your Nosana node with just a double click from the PC you wish to monitor with.
Create Monitoring .bat File
Create a new text file and save it as "monitor_nosana.bat" (change file type to "All Files")
Add Batch Script Content (Example)
@echo off title Nosana1 3080 Loc 1 echo Connecting to Nosana1 3080 Loc 1... echo Server: NosanaNode1@110.0.0.299 echo. echo When prompted, enter password: 1234qwer echo. ssh "NosanaNode1@110.0.0.299" "docker logs -f nosana-node" pause
Copy this content into your .bat file. Replace the username, IP address, and password with your actual SSH credentials. Keep the quotation marks that you see in the example they are needed
Save and Test .bat File
Save the file and double-click to run. It will connect via SSH and show live Nosana node logs

RAM Requirements
